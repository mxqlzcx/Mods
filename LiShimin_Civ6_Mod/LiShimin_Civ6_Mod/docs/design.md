# 设计文档

## 概念与愿景

本模组为《文明6》设计一个**双形态领袖**——李世民，通过"玄武门之变"这一标志性历史事件，实现领袖从武将到帝王的气质转变，让玩家在游戏中体验"马上打天下"与"下马治天下"的双重乐趣。

## 设计理念

### 1. 历史还原优先

- 严格遵循史书记载的时间线和事件
- 玄武门之变发生于唐高祖武德九年（公元626年），对应中世纪早期
- 李世民的双重身份：天策上将（军事领袖）→ 唐太宗（治世明君）

### 2. 游戏性优先

- 形态转换必须带来显著的玩法变化，而非单纯数值变更
- 天策上将形态强调军事征服和伟人积累
- 唐太宗形态强调全面发展与外交胜利
- 藩王线作为失败分支，提供重来机会而非直接终止游戏

### 3. 平衡性考量

- 两个形态各有优劣，鼓励多周目探索
- 藩王线惩罚明显但有翻盘空间
- 难度适中，兼顾新玩家体验和硬核玩家挑战

## 核心机制设计

### 双形态转换

**转换条件：**
- 进入中世纪时代
- 同时满足以下任一条件：
  - 累计征服 ≥5 座城市（正面路线：玄武门之变）
  - 累计征服 <5 座城市（负面路线：藩王线）

**转换流程：**
```
游戏开始 → 天策上将形态
    ↓
进入中世纪时代（自动检测）
    ↓
[征服≥5城] → 触发登基仪式 → 玄武门之变 → 唐太宗形态
[征服<5城] → 触发藩王线 → 复辟任务 → 唐太宗形态 或 游戏结束
```

### 天策上将形态

| 要素 | 设计 |
|------|------|
| 核心单位 | 天策上将（远程弓兵）、玄甲军（强力骑兵） |
| 特色能力 | 天策府十八学士：招募伟人，首都产出+6 |
| 胜利方式 | 军事征服、伟人征服 |
| 风险 | 核心领袖单位死亡则游戏失败 |

### 唐太宗形态

| 要素 | 设计 |
|------|------|
| 核心能力 | 贞观之治（全城产出+）、万国来朝（使者获取） |
| 胜利方式 | 全流派皆可（科技、文化、外交、宗教、征服） |
| 风险 | 无（已完成形态转换） |

### 藩王线

| 要素 | 设计 |
|------|------|
| 触发条件 | 进入中世纪但征服城市不足 |
| 惩罚 | 科技/文化/粮食产出 -50% |
| 任务 | 30回合内完成复辟（重新征服首都+建造玄武门+射杀李建成） |
| 失败后果 | 游戏结束 |

## 单位设计

### 天策上将

- **类型**：远程弓兵
- **费用**：120生产力
- **属性**：攻击40，射程2，生命200，移动3
- **秦王弓特性**：近战攻击加成+25
- **护卫机制**：2格范围内的友好单位获得防御加成

### 玄甲军

- **类型**：重装骑兵
- **费用**：180生产力（需马匹资源）
- **属性**：攻击35，防御25，生命150，移动4
- **冲锋加成**：对受损单位额外伤害+50%

## 数值平衡原则

1. **初期强势**：天策上将形态略强于标准领袖，鼓励快速扩张
2. **后期收敛**：唐太宗形态与标准领袖后期能力相当
3. **风险对等**：高风险（天策上将死亡即失败）对应高收益（更强初期能力）
4. **版本化配置**：所有数值通过 `balance.lua` 管理，支持热更新

## 用户体验设计

### UI/通知

- 使用游戏原生通知系统展示关键事件
- 颜色编码：红色（危险）、绿色（成功）、黄色（警告）
- 倒计时提示：每5回合提醒藩王线剩余时间

### 加载画面

- 显示模组logo和版本信息
- 展示当前形态状态和任务进度

### 引导

- 游戏开始时弹出简短说明
- 关键决策点提供明确选项提示
